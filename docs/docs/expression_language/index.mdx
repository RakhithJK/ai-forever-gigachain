# Язык выражений LangChain (LCEL) {#lang-chain-expression-language-lcel}

Язык выражений LangChain (LangChain Expression Language, LCEL) - это декларативный язык, который упрощает составления цепочек.
В основе LCEL лежит идея возможности внедрения прототипов в эксплуатационную среду без изменения кода, что актуально как для самых простых («промпт + LLM»), так и сложных многоэтапных цепочек.

Раздел содержит описание основных особенностей LCEL. 

## Поддержка потоковой передачи

При создании цепочек с помощью LCEL, некоторые из них могут достигнуть такой скорости, которая позволяет на лету обрабатывать токены, которые LLM  передает в потоковом режиме.

## Поддержка асинхронности

Цепочки, созданные с помощью LCEL, можно вызывать как с синхронным API (например, при прототипировании в Jupyter-блокнотах), так и с асинхронным API (например, на сервере [GigaServe](/ru/gigachat/sdk/gigaserve)). Это позволяет использовать один код для прототипирования и внедрения в эксплуатцию, с отличной производительностью и возможностью обработки большого количества одновременных запросов на одном сервере.

## Оптимизированное параллельное выполнение

При наличии в LCEL-цепочках шагов, которые можно выполнить параллельно (например, если при извлечении документов документов из нескольких ретриверов), для минимальной задержки это происходит автоматически как в синхронном, так и в асинхронном интерфейсах.

**Перезапуски и альтернативные пути**

Повышение надежности цепочек при их масштабировании за счет настройки перезапусков или альтернативных путей работы.

## Доступ к промежуточным результатам

При работе со сложными цепочками полезно иметь доступ к результатам промежуточных шагов до получения окончательного результата. Например, данные промежуточных шагов можно использовать для отладки цепочки или информирования пользователя. Промежуточные результаты можно транслировать в потоковом режиме, что доступно на любом сервере [GigaServe](/ru/gigachat/sdk/gigaserve).

## Схемы входных и выходных данных

Поддержка схем форматов Pydantic и JSONSchema для входных и выходных данных цепочки.
Схемы данных основываются на структуре цепочки и позволяют валидировать данные при использовании GigaServe.

## Бесшовная интеграция развертывания GigaServe

Цепочки, созданные с помощью LCEL, можно легко развернуть с помощью [GigaServe](/ru/gigachat/sdk/gigaserve).
